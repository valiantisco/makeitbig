# Build with Next + Tailwind

## Objective
Write clean, scalable, production-ready UI code.

## Rules

### Structure
- Use functional components
- Keep components small and reusable
- Avoid unnecessary state

### Styling
- Tailwind only
- No inline styles unless necessary
- Use consistent spacing scale

### Layout
- Prefer flex/grid over hacks
- Avoid deeply nested divs
- Keep markup readable

### Naming
- Clear, simple component names
- No overengineering

## Common Fixes

- Fix broken layouts
- Align spacing
- Clean up messy JSX
- Remove redundant wrappers

## Output Format

- Return full working code
- No partial snippets unless requested
- Clean and ready to paste
