# CLAUDE.md — Project Rules

## Stack
- Next.js (App Router)
- TypeScript (.ts / .tsx)
- Tailwind CSS
- Deployed via Vercel

## Code Philosophy
- Keep components clean and modular
- No unnecessary abstraction
- Prefer clarity over cleverness
- Use server/client split properly

## File Structure
- /app → routes
- /components → reusable UI
- /lib → helpers, utilities
- /public → assets (case-sensitive paths)

## UI / UX Principles
- Prioritize clarity and hierarchy over decoration
- Avoid cramped layouts — use horizontal space
- Strong spacing > more elements
- One primary action per section
- Reduce cognitive load at all times

## Design Standards
- Trust > creativity
- Clean, modern, minimal
- No “template feel”
- Typography should feel intentional, not default

## Brand (MakeItBig)
- Primary Red: #c40036
- Accent Blue: #17c1ce
- Tone: confident, simple, no fluff

## Copy Rules
- No em dashes
- No corporate language
- Write like a person, not a company
- Keep it tight and clear

## Output Expectations
When generating code:
- Give clean, production-ready code
- No unnecessary explanation unless asked
- If editing, use "find and replace" style

When designing UI:
- Think like a senior creative director
- Improve layout, not just styling
- Suggest better structure if needed
